# 第三轮审查意见 + 独立复检结果（针对 AGG_WATERFALL_RESEARCH_20260712）

审查方式：复现了你的全部关键数字（用你的 combined 特征表逐笔重算）；然后在**我们自己的独立引擎样本**上对你的过滤器做了阈值冻结的预注册复检（不同出场逻辑、更长窗口、0.30% 成本+真实资金费率）；另做了两项你没做的结构分析（横截面触发密度、BTC 语境）。

## 一、肯定的部分

1. **数字复现分毫不差**（强过滤 48/60.4%/+2.29%/PF3.00；高频组合 196/57.7%/+1.16%/PF1.93）——记账干净。
2. **LGB 失败的诚实报告**：与我们判断一致，1034 行喂不饱树模型，且真信号是低维结构（两三个特征），模型没得学。
3. **置换检验首次通过**：我在你的 val 段上构造了 17,648 个过滤器做"挑最优"置换检验，真实最优 +4.75% vs 纯噪声 90 分位 +4.25%，p≈0.02。你的方向不是运气（注意：日内聚类使该检验偏乐观，真实 p 更大一些但大概率仍显著）。
4. **方向已三次独立汇合**："分钟内持续主动卖压 + 贴住分钟低点"在你的 19 天搜索、我们的 taker 梯度测试、这次 train/val 中三次同向——这是这套数据里唯一反复出现的真实结构。

## 二、问题（按重要性）

1. **(0.64, 0.15) 是网格峰值**。邻域敏感性（sr 0.58-0.70 × cp 0.10-0.25，val 段）：均值 +0.19% ~ +2.29% 不等，你报告的组合恰好是全网格最大值。公允的"区域估计"是 **+1.0~1.5%/笔、PF 1.8~2.5**，不是 +2.29%/PF 3.0。建议改报邻域区域而非单点。
2. **集中度没报告**：强过滤 val 段利润前 3 天占 71%（05-25 一天 +47pp / 总 +110pp），去前 3 天 +0.83%/笔；高频组合前 3 天占 47%，去后 +0.67%。结论方向不变但幅度减半以上。
3. **族取舍是看着 validation 做的**（"other raw val PF 2.38 → 保留"）——高频组合的 196 笔数字部分是 val 拟合。族策略应在 train 上定死再看 val。
4. **自由度过多的痕迹**：每个族用不同 cutoff（40s/50s/59s）+ 不同阈值。建议统一成一种过滤形式，接受一点性能损失换稳健性。

## 三、我们的独立复检（阈值冻结、不调参）

样本：我们自己的 honest 引擎 768 笔（2025-06-18 ~ 2026-06-19，与你的 core5 同规则族但**出场逻辑不同**，0.30% 成本 + 真实资金费率）。

- **H1 强过滤方向复现成功**：基线 +1.03% → 过滤后 +2.02%（Δ+1.0pp），三个子期（2025H2/2026Q1/2026Q2）方向全部为正。
- **但去掉前 3 天（10-10、05-25、07-10）后：-0.29%**——提升全部来自级联日。
- **结构性发现**：过滤器不提高胜率（51%→45.6%），提高的是赔率（赢单均值 +3.4%、输单 -2.8%，级联日 +8.8~11%）。它是"真级联探测器"，不是"日常质量过滤器"。

## 四、两个新的结构分析（你可以直接用）

1. **横截面触发密度**（过去 60 分钟全市场触发数，事前可得）：全部机械利润集中在密度 ≥3 的窗口（+1.80% vs 密度<3 的 -0.22%）。**但**该窗口 2024 年是陷阱（胜率 17.8%、-1.54%）、2025 年是金矿（+5.28%）；把 3 个史诗日（2025-10-10、2021-05-19、2022-11-08）拿掉后高密度窗口是 -1.52%。
2. **BTC 语境门控是零结果**：高密度触发 99% 本来就发生在 BTC 4h ≤ -2% 时，两个变量重合，无增量判别力。

**合起来的图景**：这个策略族的 PnL 结构 = 日常磨损 ≈0 + 每季度几次中型事件日小丰收 + 每 1-2 年一次史诗级联大丰收。**不要再试图把它优化成日常正期望**——你三轮 + 我们七个构形的证据都说日常那部分的信息不在 1m/agg 价格数据里。

## 五、对你 partial-candle replay 的具体建议

1. 做成**级联模式触发**而不是全市场常开：全市场滚动触发计数 ≥3 时进入 cascade mode，此时才启用 40s/50s 提前入场 + agg 过滤；平时只发提醒不进场。你的 preclose AUC 阶梯（10s 0.79 → 50s 0.89）用在这个场景才有意义（59s 档不可操作，等到 59s 不如等收线）。
2. 崩盘日成交假设要修：|ret_30m|>15% 的入场按滑点 ×5 或标记不可成交，否则 10-10 类日子会把结果吹到不可信。
3. 报告规范：任何均值旁边给 top3 日占比 + ex-top3 均值；阈值给邻域网格不给单点。
